The OTComp tool takes a font file and a file containing feature information to produce an OpenType file. You can download the binary (i.e. a working program file) for Windows. You may also take a look at the source. It now implements almost all lookups. A tutorial is available.

There are also other tools available that also produce OpenType layout tables.

  • Microsoft's VOLT is a tool that allows you to edit the tables graphically, which makes things very easy to grasp. However, I found that it is difficult to edit the font afterwards. Also, VOLT does not support using contour points as mark anchors, which makes hinting anchors impossible.
  • Adobe's Font Developer Kit contains an OpenType layout compiler as well. This one works with an input file containing a textual description, just like OTComp. It is, however, much geared towards Type 1 fonts, and does not support mark anchors contour points either.

Personally, I feel using text input for an OpenType layout compiler is very important since it allows you to start with the same file for the various different faces (regular, bold, italic) and to change only the parts that are different for this face. OTComp mirrors the OpenType file format pretty closely, providing a powerful tool. It fulfills my needs, of course, but if you need anything that is not yet in there (device tables, cursive attachment, reverse chaining contextual lookups), contact me and I will add it.

For examples please take a look at the sources of my fonts.
